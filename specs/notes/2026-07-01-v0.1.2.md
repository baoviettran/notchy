## What shipped

- **Global shortcut** `CmdOrCtrl+Shift+N` (`tauri-plugin-global-shortcut`) opens the quick-add window.
- **Quick-add window** â€” second Tauri webview, 360Ă—200, decoration-less, hidden at startup, shown on demand. Renders the `/quick-add` SvelteKit route as an adding-machine "tape" line (amber phosphor amount, hairline rule, ledger payee, status line). Enter saves, Escape dismisses.
- **Single-line tokenizer** (`src/lib/utils/quick_parse.ts`) â€” pure; produces `{ kind, amount, payee }`. Leading `+` = income, default = expense. Delegates the numeric token (suffixes intact) to the existing locale-aware `parseAmount`, which owns `k`/`m`/`tr` expansion. Vietnamese diacritics in payees survive intact.
- **Default quick account** â€” persisted in `app_meta` under `default_quick_account_id`; a Settings picker chooses it (including a "None" option that clears the key via a new `deleteMeta`/`clearDefaultQuickAccount` helper, restoring the `accounts[0]` fallback).
- **Cross-window refresh** â€” on save, the quick-add window emits a Tauri `transaction:saved` event; the main window listens (via a DI'd `attachTransactionSavedListener` helper) and refetches its transaction list. Required because each Tauri webview is a separate JS context with its own stores.
- Only `expense` and `income` are supported in quick-add (refunds/transfers use the full form).
