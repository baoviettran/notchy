## What shipped

A desktop-only fast path to log a transaction without opening the main window. Click the tray icon (or press the global shortcut) → a small "tape" quick-add window appears → type shorthand → Enter → window hides, transaction saved, main DB updated live.

- **Tray icon** (core Tauri `tray-icon` feature) with menu: *Quick Add* · *Show Notchy* · *Quit*. Left-click opens the quick-add window.
- **Global shortcut** `CmdOrCtrl+Shift+N` (`tauri-plugin-global-shortcut`) opens the quick-add window.
- **Quick-add window** — second Tauri webview, 360×200, decoration-less, hidden at startup, shown on demand. Renders the `/quick-add` SvelteKit route as an adding-machine "tape" line (amber phosphor amount, hairline rule, ledger payee, status line). Enter saves, Escape dismisses.
- **Single-line tokenizer** (`src/lib/utils/quick_parse.ts`) — pure; produces `{ kind, amount, payee }`. Leading `+` = income, default = expense. Delegates the numeric token (suffixes intact) to the existing locale-aware `parseAmount`, which owns `k`/`m`/`tr` expansion. Vietnamese diacritics in payees survive intact.
- **Default quick account** — persisted in `app_meta` under `default_quick_account_id`; a Settings picker chooses it. Falls back to `accounts[0]` if unset.
- **Cross-window refresh** — on save, the quick-add window emits a Tauri `transaction:saved` event; the main window listens (via a DI'd `attachTransactionSavedListener` helper) and refetches its transaction list. Required because each Tauri webview is a separate JS context with its own stores.
- Only `expense` and `income` are supported in quick-add (refunds/transfers use the full form).

## Manual desktop smoke (NOT automatable — requires a human at `pnpm tauri dev`)

The tray icon, OS global shortcut, and real cross-window window show/focus are OS-level and cannot be exercised by Playwright. Before merging/releasing, run `pnpm tauri dev` and confirm:

- [ ] A tray icon is present in the OS tray.
- [ ] Left-clicking the tray icon opens the quick-add window.
- [ ] Tray menu *Quick Add* opens it; *Show Notchy* focuses the main window; *Quit* exits the app.
- [ ] `Cmd/Ctrl+Shift+N` opens the quick-add window (if the OS rejects the combo — another app owns it — the app logs and continues, does not crash).
- [ ] Typing `50k coffee` + Enter saves, the window hides, and the main transaction list shows the new row on next focus.
- [ ] Vietnamese path: typing `1.5tr lương` saves 1,500,000 with payee `lương` intact.
- [ ] `+50k salary` saves as income.
- [ ] Settings → Quick-add account picker loads the current default and persists a new choice.
- [ ] Escape dismisses the quick-add window.

## Known limitations (non-blocking, documented)

- **Settings picker has no "unset".** Once a default quick account is chosen, you can switch accounts but not revert to the `accounts[0]` fallback explicitly (no `deleteMeta` helper exists). Non-blocking: quick-add falls back to `accounts[0]` only when the meta key is absent, which is the case until the user first picks one.
- **Web quick-add is a separate v0.2 effort.** Tray/global-shortcut are desktop-only; the browser `/quick-add` route exists and is exercised by E2E, but there's no web UI affordance to reach it in the shipped app.
