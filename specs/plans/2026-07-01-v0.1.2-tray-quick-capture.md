# v0.1.2 Tray Quick-Capture Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a desktop-only tray icon + global shortcut that opens a small "tape" quick-add window, parses a single shorthand line into an expense/income transaction, saves it, and live-updates the main window.

**Architecture:** A dedicated `/quick-add` SvelteKit route rendered in a second Tauri webview (360×200, hidden at startup, shown on tray click / `CmdOrCtrl+Shift+N`). The window uses a pure tokenizer (`quick_parse.ts`) that splits the typed line into `{ kind, amount, payee }` and delegates the numeric token — suffixes intact — to the existing locale-aware `parseAmount`. Saving calls the existing `createTransaction` repo, then emits a Tauri `transaction:saved` event that the main window listens for and refetches on. Default account is persisted in `app_meta` under `default_quick_account_id`.

**Tech Stack:** SvelteKit 5 (runes), Tauri v2 (core tray + `tauri-plugin-global-shortcut`), Paraglide JS i18n (en/vi), Vitest + better-sqlite3 in-memory tests, Playwright E2E.

```typescript
import { parseAmount } from './number_parse';
import { stripControlChars } from './sanitize';
import type { Locale } from '$lib/utils/locale'; // adjust to actual Locale export
import { AppError } from '$lib/utils/error'; // adjust to actual AppError export

export interface QuickInput {
  kind: 'expense' | 'income';
  amount: number;
  payee: string | null;
}

/**
 * Parse a single quick-capture line.
 *
 * Grammar:  [+] <amount>[suffix] <payee…>
 *
 * The tokenizer does NOT expand k/m/tr — parseAmount owns suffix expansion
 * (number_parse.ts:28-34). We only consume the leading '+' (income) and split
 * the numeric token from the payee remainder.
 */
export function parseQuickInput(
  input: string,
  locale: Locale,
  currency: string = 'VND'
): QuickInput {
  const trimmed = input.trim();
  if (trimmed === '') {
    throw new AppError('invalid_amount');
  }

  const kind: 'expense' | 'income' = trimmed.startsWith('+') ? 'income' : 'expense';
  const body = trimmed.startsWith('+') ? trimmed.slice(1).trimStart() : trimmed;

  // Split on first whitespace: numeric token | payee remainder.
  const sp = body.search(/\s/);
  const numericToken = sp === -1 ? body : body.slice(0, sp);
  const payeeRaw = sp === -1 ? '' : body.slice(sp).trim();

  // parseAmount throws AppError('invalid_amount') if the token has no digits.
  const amount = parseAmount(numericToken, locale, currency);

  const payee = payeeRaw === '' ? null : stripControlChars(payeeRaw).trim();
  return { kind, amount, payee };
}
```

```rust
use tauri::{
    Manager,
    menu::{Menu, MenuItem},
    tray::{MouseButton, MouseButtonState, TrayIconBuilder, TrayIconEvent},
};

fn show_quick_add(app: &tauri::AppHandle) {
    if let Some(w) = app.get_webview_window("quick-add") {
        let _ = w.show();
        let _ = w.set_focus();
    }
}

fn show_main(app: &tauri::AppHandle) {
    if let Some(w) = app.get_webview_window("main") {
        let _ = w.show();
        let _ = w.set_focus();
    }
}

pub fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_log::Builder::new().build())
        .plugin(tauri_plugin_sql::Builder::default().build())
        .plugin(tauri_plugin_fs::init())
        .plugin(tauri_plugin_dialog::init())
        .plugin(
            tauri_plugin_global_shortcut::Builder::new()
                .with_shortcut("CmdOrCtrl+Shift+N")
                .unwrap()
                .with_handler(|app, _shortcut, event| {
                    if event.state() == tauri_plugin_global_shortcut::ShortcutState::Pressed {
                        show_quick_add(app);
                    }
                })
                .build(),
        )
        .setup(|app| {
            let quick_add = MenuItem::with_id(app, "quick_add", "Quick Add", true, None::<&str>)?;
            let show = MenuItem::with_id(app, "show", "Show Notchy", true, None::<&str>)?;
            let quit = MenuItem::with_id(app, "quit", "Quit", true, None::<&str>)?;
            let menu = Menu::with_items(app, &[&quick_add, &show, &quit])?;

            TrayIconBuilder::with_id("main-tray")
                .icon(app.default_window_icon().unwrap().clone())
                .menu(&menu)
                .show_menu_on_left_click(false)
                .on_menu_event(|app, event| match event.id.as_ref() {
                    "quick_add" => show_quick_add(app),
                    "show" => show_main(app),
                    "quit" => app.exit(0),
                    _ => {}
                })
                .on_tray_icon_event(|app, event| {
                    if let TrayIconEvent::Click {
                        button: MouseButton::Left,
                        button_state: MouseButtonState::Up,
                        ..
                    } = event
                    {
                        show_quick_add(app);
                    }
                })
                .build(app)?;

            Ok(())
        })
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

> **Notes for the implementer (verify against the pinned versions in Cargo.lock):**
> - `tauri-plugin-global-shortcut` v2 `Builder` API: `.with_shortcut("...")?.with_handler(...).build()` returns a plugin passed to `.plugin(...)`. If the pinned version uses a different shape (e.g. `.with_shortcuts(["..."])?` or `app.global_shortcut().register(...)` in setup), follow that version's docs — the **requirement** is: pressing `CmdOrCtrl+Shift+N` calls `show_quick_add(app)`. Do not crash if the OS rejects the combo (another app owns it) — log and continue.
> - `event.state()` may be a field (`event.state`) on some patch versions. Use whichever compiles; the pressed-edge check is the intent.
> - If `default_window_icon()` returns `None` in your config, supply an explicit `Image` from the icon path via `tauri::image::Image::from_path(...)`.

```svelte
    ready = true;

    queueMicrotask(() => document.getElementById('qa-input')?.focus());
  });

  async function submit() {
    error = null;
    const db = dbStore.db;
    if (!db) return;
    let parsed;
    try {
      parsed = parseQuickInput(value, settings.locale, settings.currency);
    } catch (e) {
      error = e instanceof AppError ? m.forms_amount_placeholder() : 'Error';
      return;
    }

    const id = await getDefaultQuickAccount(db);
    const accounts = await listAccounts(db);
    const account = (id && accounts.find((a) => a.id === id)) || accounts[0];
    if (!account) {
      error = 'No account';
      return;
    }

    await createTransaction(db, {
      kind: parsed.kind,
      date: new Date().toISOString().slice(0, 10),
      amount: parsed.amount,
      account_id: account.id,
      payee: parsed.payee,
      description: null,
      tag_id: null,
    });

    await emit('transaction:saved', { accountId: account.id });

    value = '';
    await getCurrentWindow().hide();
  }

  function onKeydown(e: KeyboardEvent) {
    if (e.key === 'Escape') { getCurrentWindow().hide(); }
    if (e.key === 'Enter') { e.preventDefault(); submit(); }
  }
</script>
```

## Task 7: Cross-Window Refresh (`attachTransactionSavedListener`)

The "main list updates" claim (design spec line 213) is load-bearing and must be unit-tested. We extract the wiring into a **pure, dependency-injected helper** rather than mounting the Svelte layout in jsdom. The helper takes the `listen` function and a `refetch` callback as arguments — trivially testable, and the layout wires real dependencies in `onMount`.

**Files:**
- Create: `src/lib/stores/quick-refresh.ts`

```typescript
import type { listen } from '@tauri-apps/api/event';

type Listen = typeof listen;

/**
 * Register a `transaction:saved` listener that refetches store data.
 *
 * Dependency-injected so the wiring is unit-testable without Tauri/jsdom:
 * the layout passes the real `listen` and an async `refetch` (which calls the
 * relevant stores' `load()`). Returns an `unlisten` for cleanup.
 */
export async function attachTransactionSavedListener(
  listen: Listen,
  refetch: () => Promise<void>
): Promise<() => void> {
  return listen('transaction:saved', async () => {
    await refetch();
  });
}
```

## Open question resolved (recorded for reviewers)

The design spec (line 169) left open which layer expands `k`/`m`/`tr`. **Resolution:** `parseAmount` (`src/lib/utils/number_parse.ts:28-34`) already expands all three, locale-aware. The tokenizer therefore does **not** touch suffixes — it consumes only the leading `+` and splits the numeric token from the payee, handing the token (suffix intact) to `parseAmount`. This is encoded in Task 1's test (`1.5tr lương` → 1,500,000 via the `tr` path) and implementation.
